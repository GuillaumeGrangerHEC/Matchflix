MATCHFLIX - Comment demarrer l'application
============================================

OPTION RAPIDE (recommandee)
----------------------------
Double-clique sur "start.bat" a la racine du projet (C:\Guillaume\start.bat).
Deux fenetres noires s'ouvrent (serveur + client), et le navigateur
s'ouvre automatiquement sur http://localhost:5173 apres quelques secondes.

Si la page ne charge pas du premier coup, attends quelques secondes
et actualise (F5) - le serveur n'a peut-etre pas fini de demarrer.

Laisse les deux fenetres noires ouvertes pendant que tu utilises l'app.
Pour tout arreter : ferme les deux fenetres (ou Ctrl+C dans chacune).


OPTION MANUELLE
----------------
Si tu preferes lancer toi-meme, ouvre deux fenetres cmd :

Fenetre 1 :
  cd /d C:\Guillaume\server
  npm run dev

Fenetre 2 :
  cd /d C:\Guillaume\client
  npm run dev

Puis ouvre http://localhost:5173 dans ton navigateur.


COMMENT TESTER UN MATCH
--------------------------
Il faut deux utilisateurs pour faire un "match". Le plus simple en local :
ouvre DEUX onglets de navigateur sur http://localhost:5173.

  1. Onglet 1 : clique "Creer", choisis un pays, Films ou Series, et
     des plateformes de streaming. Note le code a 4 chiffres affiche.
  2. Onglet 2 : clique "Rejoindre", entre ce code, choisis des
     plateformes (au moins une en commun) et eventuellement un genre.
  3. Une fois les deux prets, swipe a droite ("J'aime", ou le bouton
     coeur) sur le MEME film/serie des deux cotes pour declencher
     l'ecran de Match.

Pour voir l'app en format mobile dans le navigateur (sans telephone) :
F12 puis Ctrl+Shift+M.


TESTER SUR TON TELEPHONE (apres avoir pousse les changements)
-----------------------------------------------------------------
Le developpement se fait en local (rapide, pas d'attente de deploiement),
mais pour un vrai test sur telephone, il faut la version en ligne :

  https://matchflix-seven.vercel.app

Cette URL se mets a jour automatiquement apres chaque "git push" -
demande-moi quand tu veux pousser les changements valides en local.


A SAVOIR
----------------
- Le client (frontend) recharge automatiquement quand le code change.
- Le serveur recharge aussi automatiquement (tsx watch) depuis qu'on a
  deplace le projet sur ce disque local (C:\Guillaume).
